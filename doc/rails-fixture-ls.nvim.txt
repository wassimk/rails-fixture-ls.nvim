1. rails-fixture-ls.nvim         *rails-fixture-ls.nvim-rails-fixture-ls.nvim*

An in-process Neovim LSP server for Rails test fixture completion, hover, and
go-to-definition. Runs entirely as Lua inside your Neovim session with no
external process.

П�� HOW IT WORKS*rails-fixture-ls.nvim-rails-fixture-ls.nvim-🔍-how-it-works*

The server uses Neovim’s in-process LSP support, passing a Lua function as
the `cmd` parameter instead of spawning an external process. It only activates
in projects that have a `test/fixtures/` or `spec/fixtures/` directory under
the project root (`Gemfile`, `Rakefile`, or `.git`). If no fixture directory
exists, the server does not start.

Fixture YAML files are parsed in a single pass with simple pattern matching.
Data is cached per project root, so multi-project workspaces are fully
supported. Files using ERB (`<%= %>` tags) work correctly since ERB only
appears in attribute values, not fixture names.
